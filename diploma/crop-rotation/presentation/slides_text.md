## 2. Постановка задачи

Задача работы состоит в том, чтобы по истории использования конкретного поля и базовым признакам поля и региона сформировать рекомендацию следующей культуры.

Формально вход можно представить так:

```text
(history_1, history_2, history_3, x) -> target
```

где `history_1`, `history_2`, `history_3` — культуры за три предыдущих года, `x` — признаки поля и региона, а `target` — культура следующего года.

Практически работа решает не только задачу top-1 classification. Более полезный формат — Top-k shortlist вероятных культур, который можно анализировать с учётом confidence и explainability.

---

## 3. Проблемы аналогов и отличие работы

Существующие подходы можно условно разделить на три группы.

1. Полные оптимизационные системы севооборота решают более широкую задачу планирования и требуют большого числа ограничений и внешних параметров.
2. ML crop recommendation модели часто дают предсказание, но не всегда показывают, насколько ему можно доверять и как его интерпретировать.
3. Крупные farm management платформы покрывают много аграрных процессов, но не являются узким воспроизводимым модулем next-crop recommendation по истории конкретного поля.

Моя работа занимает промежуточную позицию: она не строит полный оптимизатор, а формирует объяснимый рекомендательный прототип. В нём есть предиктивная модель, Top-k recommendation, confidence layer, knowledge graph explainability layer и spatial demo.

---

## 4. Архитектура решения

Архитектура построена как последовательный notebook-first pipeline с сохранением промежуточных артефактов.

Основная схема:

```text
USDA NASS CSB/CDL
-> data preparation
-> window dataset
-> predictive core
-> Top-k recommendation
-> confidence analysis
-> graph-based explainability
-> spatial demo
```

Предиктивное ядро отвечает на вопрос, какие культуры наиболее вероятны. Recommendation layer переводит вероятности в shortlist. Confidence layer показывает надёжность результата. Explainability layer добавляет semantic context. Spatial demo показывает, как результат может выглядеть на карте полей.

---

## 5. Данные

В работе используются данные USDA NASS CSB/CDL за 2017-2024 годы. CDL даёт годовые классы культур, а CSB-слой используется как источник идентификаторов полей, площади и пространственного контекста.

После очистки исходный prepared dataset содержит 8 110 870 строк. Затем данные преобразуются в оконную постановку:

```text
(history_1, history_2, history_3) -> target
```

Полный window dataset содержит 40 554 350 строк. После фильтрации редких target-классов остаётся 38 511 210 строк и 9 целевых классов: `corn`, `cotton`, `fallow`, `forage_hay`, `legumes`, `other_cereals`, `sorghum`, `soybeans`, `wheat`.

Разделение train/validation/test выполнено group-aware по `CSBID`, чтобы окна одного поля не попадали одновременно в разные части выборки.

---

## 6. Baseline-модели

Для сравнения использовались несколько baseline-подходов.

`most_frequent` всегда выбирает самый частый класс в train.  
`last_crop` повторяет последнюю культуру в истории поля.  
`Markov-1` использует статистику одношаговых переходов:

```text
P(target | history_3)
```

Эти модели нужны как честные ориентиры. Они показывают, насколько качество финальной модели связано с использованием более содержательной истории и контекстных признаков, а не только с частотами или инерцией последовательности.

На test `Markov-1` даёт Accuracy 0.557219, Macro F1 0.411371 и Weighted F1 0.539027.

---

## 7. Модель предсказания

Финальной моделью диплома является baseline CatBoost.

Она использует компактный набор признаков:

- `history_1`, `history_2`, `history_3`;
- `STATEFIPS`, `CNTYFIPS`;
- `CSBACRES`;
- `INSIDE_X`, `INSIDE_Y`.

CatBoost выбран потому, что задача имеет табличную природу и сочетает категориальные и числовые признаки. Модель возвращает вероятности по всем целевым классам, а дальше эти вероятности используются для формирования Top-k shortlist.

На test baseline CatBoost показывает:

- Accuracy = 0.699006;
- Macro F1 = 0.568485;
- Weighted F1 = 0.689626.

---

## 8. Промежуточные эксперименты

Помимо финального baseline CatBoost были проверены две экспериментальные ветки.

**CatBoost + extra features** добавлял derived-признаки истории: число уникальных культур, наличие `fallow` и `legumes`, признаки повторяемости. На test качество оказалось близким: Accuracy = 0.699235, Macro F1 = 0.569473, Weighted F1 = 0.689991. Несмотря на небольшой прирост, ветка сохранена как experimental non-final, потому что она не меняет итоговый проектный контракт.

**CatBoost + transition graph** проверял гибридное ранжирование:

```text
score = alpha * p_catboost + beta * p_graph
```

Лучший вариант с `alpha = 0.9`, `beta = 0.1` на test уступил чистому CatBoost: Accuracy = 0.698714, Macro F1 = 0.564387, Weighted F1 = 0.688729.

Итог: baseline CatBoost выбран как финальное предиктивное ядро, а graph-сигнал перенесён в explainability layer.

---

## 9. Recommendation results

Главная прикладная постановка работы — не один ответ, а shortlist вероятных следующих культур.

Для оценки используются:

- `Accuracy@1`;
- `Hit@3`;
- `Hit@5`.

На test CatBoost показывает:

- Accuracy@1 = 0.699006;
- Hit@3 = 0.951236;
- Hit@5 = 0.990819.

Для сравнения, Markov-1 на test:

- Accuracy@1 = 0.557219;
- Hit@3 = 0.855303;
- Hit@5 = 0.931717.

Это показывает, что CatBoost полезен не только как классификатор, но и как shortlist-рекомендатель: правильная культура почти всегда попадает в первые 3-5 кандидатов.

---

## 10. Confidence layer

После формирования shortlist важно понять, насколько устойчиво выглядит рекомендация.

Confidence layer использует вероятности CatBoost и делит случаи на три группы:

- `high_confidence`;
- `medium_confidence`;
- `low_confidence`.

На test результаты такие:

| Группа | Наблюдений | Accuracy@1 | Hit@3 |
|---|---:|---:|---:|
| high_confidence | 3 042 231 | 0.84147 | 0.97713 |
| medium_confidence | 1 691 260 | 0.60931 | 0.94252 |
| low_confidence | 1 043 523 | 0.42902 | 0.88987 |

Вывод: confidence-сигнал действительно разделяет более устойчивые и более спорные рекомендации. Даже в low-confidence группе модель остаётся полезной как shortlist, но top-1 результат нужно интерпретировать осторожнее.

---

## 11. Калибровка

Для проверки confidence-сигнала дополнительно анализировалась калибровка вероятностей на preview-выборках по 5000 объектов.

Для test получены:

- ECE = 0.01737;
- Brier score = 0.16120;
- mean confidence = 0.75398;
- accuracy@1 = 0.74800.

Reliability diagram показывает, насколько средняя уверенность модели согласована с фактической точностью внутри интервалов вероятности.

Эти результаты не означают идеальную калибровку во всех областях, но подтверждают, что вероятность модели можно использовать не только для сортировки кандидатов, но и как содержательный confidence-сигнал.

---

## 12. Explainability layer

Explainability layer построен на основе knowledge graph и набора интерпретационных правил.

Важно: knowledge graph не заменяет CatBoost и не является второй предиктивной моделью. Он работает поверх уже сформированного shortlist:

```text
model shortlist -> graph/rule context -> candidate-level explanation
```

Граф содержит культуры, группы культур, регионы и правила. По метаданным запуска: 2827 узлов, 18154 ребра и 5 интерпретационных правил.

Для каждого кандидата из shortlist формируются:

- model score;
- transition support;
- regional signal;
- support/warning flags;
- `graph_support_score`;
- краткое объяснение.

---

## 13. Knowledge graph rules

В explainability layer используются пять правил.

| Правило | Тип сигнала | Смысл |
|---|---|---|
| `repeat_crop_warning` | warning | предупреждение при повторе последней культуры |
| `same_group_saturation` | warning | предупреждение при насыщении истории одной группой |
| `legume_break_bonus` | support | поддержка бобовых, если их не было в недавней истории |
| `fallow_break_effect` | support / warning | интерпретация выхода из `fallow` или повторного `fallow` |
| `region_typicality` | support | поддержка культур, типичных для региона |

Эти правила не являются полной экспертной агрономической системой. Их роль — дать понятные semantic clues для анализа shortlist-кандидатов.

## 14. Case study

Case studies показывают, как система работает на уровне конкретных наблюдений.

Были рассмотрены четыре типа случаев:

- high confidence;
- medium confidence;
- low confidence;
- difficult / disagreement case.

Примеры:

- high confidence: фактическая культура `forage_hay`, top-1 CatBoost тоже `forage_hay` с `p = 0.930`, KG также сильнее всего поддерживает `forage_hay`;
- medium confidence: фактическая культура `corn`, top-1 верен, но KG сильнее поддерживает альтернативу `soybeans`;
- low confidence: фактическая культура `legumes`, top-1 верен, но вероятность всего `p = 0.432`, а KG сильнее поддерживает `soybeans`;
- difficult case: фактическая культура `soybeans`, top-1 модели `corn`, но `soybeans` остаётся в Top-3 и получает более сильную KG-поддержку.

Вывод: полезность системы проявляется не только в правильном top-1, но и в возможности анализировать shortlist и конкурирующие альтернативы.

## 15. Spatial demo

Spatial demo показывает, как recommendation и explainability results можно связать с конкретными полями на карте.

Этот слой не является новой моделью и не влияет на метрики. Его задача — аналитическая и интерфейсная демонстрация.

В проекте сформированы spatial artifacts, включая:

- `spatial_explainable_demo_map.html`;
- `spatial_bbox_overview_map.html`;
- `spatial_bbox_demo_map.html`;
- `spatial_layer_comparison_summary.csv`;
- `spatial_bbox_selection_summary.csv`.

В spatial context полный региональный слой содержит 19 919 полей. После фильтрации до рабочего пространства из 9 target-классов остаётся 16 663 поля, то есть около 83.7% полного слоя. Для компактной demo-области через bounding box выбран clickable dataset из 23 полей.

## 16. Итоговые выводы

В работе построен воспроизводимый прототип next-crop recommendation system.

Основные результаты:

- подготовлены данные USDA NASS CSB/CDL и оконная выборка `history -> target`;
- зафиксирован group-aware split по `CSBID`;
- baseline CatBoost выбран как финальная модель;
- показано преимущество CatBoost над простыми baseline и Markov-1;
- задача переведена из top-1 classification в Top-k recommendation;
- добавлен confidence layer;
- построен knowledge graph explainability layer;
- сформирована spatial demo для визуального анализа результатов.

Главный вывод: задача выбора следующей культуры хорошо ложится на формат explainable recommendation / decision-support prototype, где модель не заменяет эксперта, а помогает анализировать вероятные варианты.

## 17. Перспективы

Дальнейшее развитие работы может идти в нескольких направлениях.

1. Расширение признакового пространства внешними агрономическими факторами: почва, погода, экономические параметры, ограничения хозяйства.
2. Более глубокий confidence-анализ и отдельная настройка калибровки вероятностей.
3. Развитие knowledge graph: больше правил, более богатые связи между культурами и регионами.
4. Улучшение пространственного интерфейса и сценариев работы с картой.
5. Проверка переносимости подхода на другие регионы и временные интервалы.

При этом важно сохранить текущую рамку: система остаётся инструментом поддержки принятия решений, а не гарантированным оптимизатором севооборота.
