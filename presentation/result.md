# Описание дипломной работы

## Тема и цель исследования

**Тема:** рекомендательная система по севообороту на основе историй культур по полям.

**Цель работы:** построить воспроизводимый исследовательский ML-пайплайн, который по последовательности культур за предыдущие годы и дополнительным признакам поля/региона рекомендует наиболее вероятную следующую культуру.

Задача формализована как **многоклассовая классификация** и одновременно как **задача Top-k рекомендаций**:
- вход: история по полю (`history_1`, `history_2`, `history_3`) + контекстные признаки;
- выход: класс следующей культуры (`target`) и ранжированный список кандидатов (Top-3, Top-5).

## Что было сделано

### 1) Подготовка данных и формирование исследовательского контура

В проекте реализован модульный пайплайн ноутбуков (`01` -> `09`) с сохранением промежуточных артефактов в `artifacts/`:
- подготовка и очистка данных;
- построение оконного датасета;
- фиксированный split train/val/test;
- обучение базовых и расширенных моделей;
- оценка рекомендаций;
- графовые и explainability-эксперименты.

Это обеспечивает воспроизводимость и независимый перезапуск этапов.

### 2) Источник данных и структура выборки

Использованы данные CDL по годам (2017-2024) и дополнительные признаки поля/региона.

Ключевые размеры после этапов подготовки:
- подготовленный датасет после базовой очистки: **8,110,870** строк (`artifacts/data/01_meta.json`);
- оконный датасет (history->target): **40,554,350** строк;
- после фильтрации редких target-классов: **38,511,210** строк (`artifacts/data/02_meta.json`).

В окне использовано преобразование:
- `history_1, history_2, history_3 -> target`.

### 3) Работа с классами культур

Была реализована агрегация и фильтрация целевых классов.

Из 18 target-классов удалены редкие (порог доли `< 1%`), в итоговой постановке осталось **9 классов**:
- corn
- cotton
- fallow
- forage_hay
- legumes
- other_cereals
- sorghum
- soybeans
- wheat

### 4) Формирование признаков

#### Базовый набор признаков (baseline CatBoost)
- `history_1`, `history_2`, `history_3`
- `STATEFIPS`, `CNTYFIPS`
- `CSBACRES`
- `INSIDE_X`, `INSIDE_Y`

#### Улучшенный набор признаков (improved CatBoost)
Дополнительно добавлены derived-признаки структуры истории:
- `n_unique_history`
- `has_fallow_in_history`
- `has_legume_in_history`
- `same_h1_h2`
- `same_h2_h3`
- `same_all_history`

### 5) Разделение на выборки

Сформирован фиксированный split с `random_state=42`:
- train: **70%**
- validation: **15%**
- test: **15%**

Фактические размеры (`artifacts/research_checkpoint/baseline_meta.json`):
- train: **26,957,630**
- validation: **5,776,566**
- test: **5,777,014**

### 6) Обучение и сравнение алгоритмов

В дипломе реализованы и сопоставлены несколько подходов:

1. **CatBoost baseline** (основная вероятностная модель).
2. **CatBoost improved** (расширенный набор признаков, подготовлен как отдельный этап).
3. **Гибрид CatBoost + transition graph** (reranking через матрицу переходов культур).
4. **Markov-1 baseline** (чисто переходная вероятностная модель по последней культуре).
5. **Knowledge Graph explainability layer** (прозрачный rule-based слой объяснимости без замены основной модели).

## Основные результаты

### A. Качество CatBoost baseline (классификация)

По строкам `model=catboost` в файле сравнения `artifacts/results/graph_hybrid/hybrid_metrics.csv`:

**Validation (CatBoost):**
- Accuracy: **0.69928**
- Macro F1: **0.56877**
- Weighted F1: **0.68997**

**Test (CatBoost):**
- Accuracy: **0.69901**
- Macro F1: **0.56849**
- Weighted F1: **0.68963**

Итого: получена устойчивая baseline-модель на большом реальном датасете с приемлемым балансом точности и устойчивости по классам.

### B. Качество рекомендаций CatBoost (Top-k)

По `artifacts/results/recommendation/recommendation_metrics_baseline.csv`:

**Validation:**
- Accuracy@1: **0.69928**
- Hit@3: **0.95119**
- Hit@5: **0.99079**

**Test:**
- Accuracy@1: **0.69901**
- Hit@3: **0.95124**
- Hit@5: **0.99082**

Итого: в рекомендательном режиме модель даёт очень высокий охват в Top-3/Top-5.

### C. Гибрид CatBoost + граф переходов

Тестировалась смесь вероятностей:
- `score = alpha * p_catboost + beta * p_graph`,
- сетка: `(alpha, beta) = (0.9,0.1), (0.8,0.2), (0.7,0.3)`.

Лучший конфиг: **alpha=0.9, beta=0.1** (`best_hybrid_config.json`).

Результат на test:
- Accuracy: **0.69871**
- Macro F1: **0.56439**
- Weighted F1: **0.68873**
- Hit@3: **0.94956**

Вывод: в текущей конфигурации гибрид немного уступил чистому CatBoost, то есть графовый reranking не дал прироста качества метрик, но подтвердил корректность интеграции графового сигнала.

### D. Markov-1 baseline (рекомендации)

По `artifacts/results/markov_recommendation/overall_metrics.csv`:

**Validation:**
- Accuracy@1: **0.55714**
- Hit@3: **0.85530**
- Hit@5: **0.93178**

**Test:**
- Accuracy@1: **0.55722**
- Hit@3: **0.85530**
- Hit@5: **0.93172**

Сравнение с CatBoost показывает заметный отрыв CatBoost по всем метрикам. Это подтверждает, что учет дополнительных признаков и нелинейных зависимостей существенно улучшает рекомендации.

### E. Анализ уверенности рекомендаций (confidence buckets)

По `artifacts/results/recommendation/confidence_summary_baseline.csv` (test):
- **high_confidence**: Accuracy@1 = **0.84147**, Hit@3 = **0.97713**
- **medium_confidence**: Accuracy@1 = **0.60931**, Hit@3 = **0.94252**
- **low_confidence**: Accuracy@1 = **0.42902**, Hit@3 = **0.88987**

Это позволяет выделять зоны надёжных и рискованных рекомендаций и задавать пороги для практического использования.

### F. Explainability через Knowledge Graph

Построен слой графовой интерпретации результатов (`09_knowledge_graph_explainability.ipynb`):
- узлы: культуры, регионы, группы, правила;
- рёбра: переходы, принадлежность к группе, региональные связи, rule-links;
- итоговые размеры графа (`run_meta.json`):
  - nodes: **2,827**
  - edges: **18,154**
  - rules: **5**
  - case studies: **4**

Реализованные rule-типы:
- repeat crop warning
- same group saturation
- legume break bonus
- fallow break effect
- region typicality

Для кейсов считается `graph_support_score` и тип согласованности top-1 с правилами (`strong`/`weak`), что повышает прозрачность выдачи без изменения базовой модели.

## Практическая ценность

1. Получен полностью воспроизводимый исследовательский контур для задачи рекомендаций в севообороте.
2. Показано, что CatBoost даёт сильный baseline на десятках миллионов обучающих примеров.
3. Подтверждена полезность Top-k постановки: даже при умеренном Accuracy@1 достигается очень высокий Hit@3/Hit@5.
4. Добавлен explainability-слой на базе knowledge graph и правил, который делает рекомендации интерпретируемыми для агрономического контекста.
5. Проведено сравнение с более простыми вероятностными подходами (Markov), что даёт обоснование выбора основной модели.

## Ограничения текущей версии

1. Гибрид CatBoost+Graph в текущих весах не улучшил baseline-метрики.
2. На редких классах качество ниже, чем на массовых (типичная проблема дисбаланса).
3. Разделение выполнено как долевое; для отдельных сценариев может потребоваться более строгий group-aware split по идентификатору поля.
4. Explainability-слой пока носит rule-based характер и не является причинной моделью.

## Итоговый вывод

В рамках дипломной работы построена и проверена end-to-end система рекомендации следующей культуры по истории поля. Основная модель CatBoost продемонстрировала стабильные результаты (около **0.699 Accuracy@1** и **0.951 Hit@3** на test), а дополнительные этапы (Markov-сравнение, graph reranking, knowledge-graph explainability) обеспечили методическую полноту исследования: от сильного baseline до интерпретации и анализа ограничений.
